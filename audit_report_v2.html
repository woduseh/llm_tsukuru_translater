A. 개요 및 기술 스택

RPG Maker MV/MZ 및 Wolf RPG Editor 게임의 텍스트를 추출·번역·적용하는 Electron 데스크톱 앱. Gemini LLM API를 통한 자동 번역, 이미지/오디오 복호화, JSON 구조 검증 및 복구 기능을 포함한다. TypeScript 기반의 메인-렌더러 IPC 아키텍처로, 약 51개 소스 파일, 7,500줄 이상으로 구성된다.

Electron 35.x TypeScript 5.x (strict) Node.js 20 Vitest 4.x ESLint + @typescript-eslint electron-store (암호화) axios Gemini API js-yaml lodash SweetAlert2 iconv-lite electron-builder GitHub Actions CI

C. 핵심 발견사항

1. 지속가능성 5.5

✅ 모듈 분리 완료: main.ts가 750줄에서 37줄로 축소. 6개 IPC 핸들러 모듈(src/ipc/)로 관심사 분리. 새 엔지니어가 특정 기능만 찾아 수정 가능.
⚠️ globalThis 잔존 57건+: apply.ts(9회), translator.ts(27회), fileCrypto.ts(8회) 등이 여전히 globalThis.mwindow.webContents.send() 직접 사용. AppContext 도입했으나 마이그레이션 미완.
⚠️ any 타입 248건: strict 모드 활성화로 null 안전성은 확보했지만, ESLint 기준 no-explicit-any 경고 248건. 실질적 타입 안전성에 구멍.

2. 재현성 6.5

✅ CI/CD 파이프라인 구축: GitHub Actions(.github/workflows/ci.yml)에서 tsc + ESLint + vitest 자동 실행. package-lock.json 추적으로 의존성 고정.
⚠️ 테스트 커버리지 15% 미만: 51개 소스 파일 중 3개 모듈만 테스트(globalutils, edtool, verify). 핵심인 extract, apply, translator는 테스트 없음. 통합 테스트 부재.
⚠️ Windows 전용: CI가 windows-latest만 타겟. npm start에 chcp 65001(Windows 코드페이지). 크로스 플랫폼 검증 없음.

3. 보안 5.5

✅ Electron 보안 모델 적용: contextIsolation: true + preload 스크립트 + 채널 화이트리스트(21 send, 15 receive). nodeIntegration: false로 렌더러에서 직접 Node 접근 차단.
⚠️ preload의 nodeFs 노출: src/preload.ts가 window.nodeFs.readFileSync, readdirSync, existsSync를 무제한 경로로 노출. 렌더러 XSS 시 임의 파일 읽기 가능.
❌ extentions.ts 무검증 실행파일 다운로드: 외부 URL에서 바이너리 다운로드 후 체크섬/서명 검증 없이 실행. 공급망 공격 벡터.

4. 코드 품질 5.5

✅ TypeScript strict + ESLint 0 error: tsconfig.json에 strict: true, 빈 catch 블록 전부 수정, 중복 유틸리티 통합 완료.
⚠️ 대형 파일 잔존: extract.ts(577줄), renderer.ts(618줄), llm-compare/script.ts(580줄), verify.ts(517줄) — 각각 파싱·포맷·파일I/O·UI가 혼재.
⚠️ innerHTML 15건: llm-compare/script.ts(7건), json-verify/script.ts(8건)에서 innerHTML 직접 조작. 현재 사용자 입력 직접 삽입은 없으나 잠재적 XSS 경로.
